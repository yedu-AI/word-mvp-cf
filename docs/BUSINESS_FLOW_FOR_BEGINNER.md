# 业务流程图（非技术同学版）

## 1. 你正在搭建什么
一个能持续迭代的在线产品，不是一次性页面。  
所以必须同时有：
1. 前端页面（用户看得到）
2. 后端 API（处理业务规则）
3. 数据库（保存用户数据）

## 2. 每一步在业务上的意义
1. GitHub 仓库  
意义：团队协作和版本管理中心。

2. Cloudflare Pages  
意义：用户访问的网站入口。

3. Cloudflare Worker API  
意义：处理登录、任务分配、学习记录等业务逻辑。

4. D1 数据库  
意义：保存账号、词库、学习进度，保证数据不丢。

5. `VITE_API_BASE` 环境变量  
意义：告诉前端“后端地址在哪”，否则页面无法调接口。

6. 自动化脚本  
意义：减少人工失误，提升搭建速度，换项目可复用。

## 3. 最小可用闭环（MVP 基建）
用户打开网站 -> 登录 -> 拉取今日任务 -> 提交反馈 -> 数据写入数据库  
这条链打通，就可以开始做业务细节。

## 4. 你和 AI 如何配合最省力
1. 你负责“授权、确认、截图”
2. AI 负责“写代码、部署、联调、修复”
3. 所有状态写入 `STATE.json`，换 AI 也能继续
